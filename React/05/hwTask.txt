Особливості рендерінгу в React:

Декларативність: React дозволяє описати, як компонент або елемент повинен виглядати в будь-який момент часу, і автоматично змінює DOM для відповідності цьому опису.

Віртуальний DOM: React використовує концепцію віртуального DOM, який є легковаговою копією реального DOM. При зміні стану компонента React спочатку вносить зміни в віртуальний DOM, порівнює його з попередньою версією, і потім мінімально оновлює реальний DOM.

Компонентний підхід: Рендеринг у React організований навколо компонентів, що дозволяє розбивати інтерфейс на незалежні, перевикористовувані частини.

Додавання key до елементів під час рендеру списку:

Коли рендерите список елементів у React, важливо вказати унікальний key для кожного елемента списку. Key допомагає React ідентифікувати, які елементи змінилися, додані або видалені, що значно покращує продуктивність оновлення списку.
Унікальність: Кожен key повинен бути унікальним серед сусідніх елементів у списку.

Стабільність: Ключі повинні бути стабільними, тобто не змінюватися при перерендерингу.
